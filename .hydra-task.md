# Task: Default codex agent to full-auto mode

## Problem
When spawning a codex worker via `hydra-worker`, codex launches in "suggest" mode which requires manual approval for every command. This defeats the purpose of autonomous workers.

## What to change

In `src/utils/agentConfig.ts`, the default agent command for codex is just `'codex'`. Change it to `'codex --full-auto'` so codex workers run without requiring manual approvals.

Look at `getAgentCommand()` function — the default commands map has `codex: 'codex'`. Change it to `codex: 'codex --full-auto'`.

## Acceptance criteria
- Default codex command is `codex --full-auto`
- `npm run compile` passes
- No other agent commands are changed
