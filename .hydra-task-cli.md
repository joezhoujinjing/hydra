# Task: Rewrite Hydra CLI as TypeScript with Commander.js

## Overview

Rewrite the bash Hydra CLI (`/opt/homebrew/bin/hydra`) as TypeScript by:
1. Extracting a `src/core/` layer with pure Node.js code (no `vscode` imports)
2. Building a CLI using Commander.js that imports from `src/core/`
3. Updating `src/utils/` to re-export from `src/core/`

The goal is to fix the **session naming mismatch** — the bash CLI uses `<repo>_<slug>` while the extension uses `<repo>-<hash>_<slug>` (via `getRepoSessionNamespace()`). By sharing code, both paths produce identical session names.

/** Build the shell command to launch an agent (matches bash CLI get_agent_command) */
export function buildAgentLaunchCommand(
  agentType: string,
  agentBinary: string,
  task?: string,
  repoRoot?: string
): string {
  switch (agentType) {
    case 'claude': {
      let flags = '--dangerously-skip-permissions';
      if (repoRoot) flags += ` --add-dir ${repoRoot}`;
      return task ? `${agentBinary} ${flags} -- ${shellQuoteForDisplay(task)}` : `${agentBinary} ${flags}`;
    }
    case 'codex':
      return task ? `${agentBinary} --full-auto ${shellQuoteForDisplay(task)}` : `${agentBinary} --full-auto`;
    case 'gemini':
      return task ? `${agentBinary} -y --include-directories /tmp ${shellQuoteForDisplay(task)}` : `${agentBinary} -y --include-directories /tmp`;
    case 'aider':
      return agentBinary;
    default:
      return agentBinary;
  }
}

function shellQuoteForDisplay(s: string): string {
  // Use the shellQuote from core/shell if needed, or just quote for display
  return `'${s.replace(/'/g, "'\\''")}'`;
}

## Phase 1.5: Create `src/core/sessionManager.ts`

This is the **central piece** — a shared SessionManager that both CLI and extension use for the full worker/copilot lifecycle. It uses `sessions.json` as persistent state and reconciles with live tmux on every read.

### SessionManager Design

```typescript
import * as fs from 'fs';
import * as path from 'path';
import * as os from 'os';
import { MultiplexerBackendCore, HydraRole, AgentType } from './types';
import * as coreGit from './git';
import { injectWorkerInstructions, ensureHydraGlobalConfig } from './hydraGlobalConfig';
import { buildAgentLaunchCommand, DEFAULT_AGENT_COMMANDS } from './agentConfig';
import { exec } from './exec';

const HYDRA_DIR = path.join(os.homedir(), '.hydra');
const SESSIONS_FILE = path.join(HYDRA_DIR, 'sessions.json');

// ── Types ──

export interface WorkerInfo {
  sessionName: string;
  repo: string;        // repo basename
  repoRoot: string;    // absolute path
  branch: string;
  slug: string;
  status: 'running' | 'stopped';
  attached: boolean;   // is tmux session currently attached? (tree view needs this)
  agent: string;
  workdir: string;
  tmuxSession: string;
  createdAt: string;
  lastSeenAt: string;
}

export interface CopilotInfo {
  sessionName: string;
  status: 'running' | 'stopped';
  agent: string;
  workdir: string;
  tmuxSession: string;
  createdAt: string;
  lastSeenAt: string;
}

export interface SessionState {
  copilots: Record<string, CopilotInfo>;
  workers: Record<string, WorkerInfo>;
  updatedAt: string;
}

export interface CreateWorkerOpts {
  repoRoot: string;
  branchName: string;
  agentType?: string;          // defaults to 'claude'
  baseBranchOverride?: string;
  task?: string;
  agentCommand?: string;       // override the agent binary/command (extension reads from vscode config)
}

export interface CreateCopilotOpts {
  workdir: string;
  agentType?: string;
  sessionName?: string;        // optional override
  agentCommand?: string;
}

// ── SessionManager Class ──

export class SessionManager {
  constructor(private backend: MultiplexerBackendCore) {}

  // ── Sync: reconcile sessions.json ↔ live tmux ──

  async sync(): Promise<SessionState> {
    // 1. Read sessions.json (or create empty)
    // 2. Query tmux listSessions()
    // 3. For each JSON entry:
    //    - tmux alive → status = "running", update lastSeenAt
    //    - tmux dead + worktree exists → status = "stopped"
    //    - tmux dead + no worktree → remove from JSON (orphan)
    // 4. For tmux sessions NOT in JSON but with @hydra-role metadata → add them
    // 5. Write reconciled JSON
    // 6. Return the state
  }

  async listWorkers(repoRoot?: string): Promise<WorkerInfo[]> {
    const state = await this.sync();
    const workers = Object.values(state.workers);
    return repoRoot ? workers.filter(w => w.repoRoot === repoRoot) : workers;
  }

  async listCopilots(repoRoot?: string): Promise<CopilotInfo[]> {
    const state = await this.sync();
    const copilots = Object.values(state.copilots);
    return repoRoot ? copilots.filter(c => c.workdir.startsWith(repoRoot)) : copilots;
  }

  async getWorker(sessionName: string): Promise<WorkerInfo | undefined> {
    const state = await this.sync();
    return state.workers[sessionName];
  }

  // ── Worker Lifecycle ──

  async createWorker(opts: CreateWorkerOpts): Promise<WorkerInfo> {
    // Full worker creation flow:
    // 1. Validate inputs
    // 2. getRepoSessionNamespace(repoRoot, this.backend)
    // 3. Check branch exists (resume logic)
    // 4. Detect base branch via getBaseBranchFromRepo()
    // 5. Slug collision resolution via isSlugTaken()
    // 6. addWorktree()
    // 7. Resolve @imports in instruction files
    // 8. injectWorkerInstructions()
    // 9. Create tmux session + set metadata
    // 10. Build & send agent launch command
    // 11. For claude: delayed Enter to dismiss trust prompt
    // 12. Write to sessions.json
    // 13. Return WorkerInfo
  }

  async deleteWorker(sessionName: string): Promise<void> {
    // 1. Kill tmux session if running
    // 2. Find worktree from sessions.json or by parsing session name
    // 3. removeWorktree() + delete branch
    // 4. Remove from sessions.json
  }

  async stopWorker(sessionName: string): Promise<void> {
    // 1. Kill tmux session
    // 2. Update status in sessions.json to "stopped"
  }

  async startWorker(sessionName: string, agentType?: string, agentCommand?: string): Promise<WorkerInfo> {
    // 1. Find worker in sessions.json
    // 2. Verify worktree exists
    // 3. Create tmux session in worktree dir
    // 4. Set metadata
    // 5. Launch agent
    // 6. Update sessions.json status to "running"
    // 7. Return updated WorkerInfo
  }

  // ── Copilot Lifecycle ──

  async createCopilot(opts: CreateCopilotOpts): Promise<CopilotInfo> { ... }
  async deleteCopilot(sessionName: string): Promise<void> { ... }

  // ── Private helpers ──

  private readSessionState(): SessionState { ... }
  private writeSessionState(state: SessionState): void { ... }
  private resolveImports(filePath: string, repoRoot: string): void {
    // Port of bash resolve_imports: read file, replace lines starting with @<path>
    // with contents of <repoRoot>/<path>
  }
}
```

### Key behaviors:

**Resume logic in createWorker():**
When the branch already exists:
- If tmux session is running → send task (if provided) via sendKeys, return existing info
- If worktree exists but tmux dead → create new tmux session in existing worktree, launch agent
- If branch exists but no worktree → error (tell user to delete branch)

**Session naming (THE FIX):**
Always use `this.backend.buildSessionName(repoSessionNamespace, finalSlug)` which produces `<repo-hash>_<slug>` matching the extension. This fixes the mismatch where the bash CLI used `<repo>_<slug>`.

**@imports resolution:**
Port the bash `resolve_imports` function: for each line in a file, if it starts with `@<path>`, replace it with the contents of `<repoRoot>/<path>`. Apply to CLAUDE.md, AGENTS.md, GEMINI.md in the worktree.

**Delayed Enter for Claude:**
After launching claude agent, wait ~8 seconds then send Enter to dismiss the "trust external CLAUDE.md" prompt. Use `setTimeout` + `this.backend.sendKeys()`.

## Key Design Decisions

1. **Backend is always tmux for CLI** — the CLI doesn't support zellij. Create the TmuxBackendCore directly.
2. **SessionManager is the single source of truth** — both CLI and extension use it. sessions.json is maintained by SessionManager and reconciled with live tmux on every read.
3. **CODE_DIR discovery** — for the `list` command's repo scanning, use `HYDRA_CODE_DIR` env var or default to `~/code`, same as bash script.
4. **@imports resolution** — implemented as a method in SessionManager, used during createWorker().
